# InfiniteTap

InfiniteTap은 **Apple Vision Pro(visionOS) 스위치 제어(Switch Control)** 와 연동하기 위한 **배터리리스 BLE HID 스위치 허브** 프로젝트입니다.  
기성 보조공학 스위치 인터페이스의 가격 부담과 배터리 유지보수 문제를 줄이기 위해, **M5Stamp C3U + 3.5mm 모노 스위치 입력 + USB-C 상시 전원** 조합으로 작고 단순한 동글형 장치를 만드는 것이 목표입니다.

현재 저장소에는 두 가지 스케치가 포함되어 있습니다.

- `InfiniteTap_HardwareTest`
  외부 스위치와 납땜 상태를 빠르게 확인하는 **기본 하드웨어 검증용** 스케치입니다.
- `InfiniteTap_BLE`
  BLE HID 키 입력 전송과 상태 LED, 본딩 초기화 등을 포함하는 **실사용 펌웨어 방향의 스케치**입니다.

하드웨어 조립 직후에는 반드시 `InfiniteTap_HardwareTest`부터 업로드해서 입력 배선이 정상인지 먼저 확인하는 것을 권장합니다.

## 프로젝트 목표

- Apple Vision Pro의 스위치 제어에서 사용할 수 있는 BLE HID 스위치 허브 만들기
- 배터리 없이 USB-C 전원만으로 장시간 안정 동작하기
- 적응형 스위치를 3.5mm 모노 잭으로 간단히 연결할 수 있게 하기
- 작은 보조기기 형태를 위해 보드와 소켓을 직접 납땜한 동글 구조로 구성하기

## 필요한 하드웨어

- `M5Stamp C3U` 1개
- `PJ-320` 3.5mm 모노 소켓 1개
- 외부 적응형 스위치 1개
- USB-C 전원 공급 장치 또는 USB 포트
- 점퍼선 또는 직접 납땜용 배선재
- 납땜 인두, 납, 절연 보조재

## 배선 정보

이 프로젝트는 다음 핀 구성을 기준으로 작성되어 있습니다.

- `PJ-320 GND 핀` -> `M5Stamp C3U GND`
- `PJ-320 Tip 핀` -> `M5Stamp C3U GPIO 8 (G8)`

내장 자원은 아래처럼 사용합니다.

- `내장 RGB LED` -> `GPIO 2`
- `보드 중앙 버튼` -> 펌웨어 내 본딩 초기화 입력으로 사용

## 왜 GPIO 8을 쓰나요?

초기에는 `GPIO 0`을 입력으로 고려했지만, ESP32-C3 계열에서 `GPIO 0`은 부팅 동작에 영향을 주는 스트래핑 핀입니다.  
스위치가 눌린 상태로 전원이 들어오면 다운로드 모드 진입 같은 예기치 않은 동작이 발생할 수 있으므로, 현재는 **부팅 안정성을 위해 GPIO 8로 변경**되어 있습니다.

## 개발 환경

아래 환경 중 하나를 사용하면 됩니다.

- Arduino IDE
- PlatformIO

Arduino IDE 기준으로는 다음 항목이 필요합니다.

1. `ESP32 by Espressif Systems` 보드 패키지 설치
2. 보드 선택을 `ESP32C3 Dev Module` 또는 M5Stamp C3U와 호환되는 ESP32-C3 계열 보드로 설정
3. 사용 라이브러리 설치

현재 스케치에서 사용 중인 주요 라이브러리:

- `Adafruit NeoPixel`
- `HijelHID_BLEKeyboard`  
  이 라이브러리는 일반적인 기본 설치 목록에 없을 수 있으므로, 사용 중인 개발 환경에서 별도 설치가 필요할 수 있습니다.

## 빠른 시작

### 1. 하드웨어 테스트 먼저 하기

조립 후 가장 먼저 `InfiniteTap_HardwareTest/InfiniteTap_HardwareTest.ino`를 업로드합니다.

이 스케치의 동작은 매우 단순합니다.

- 평상시: 내장 LED가 켜짐
- 외부 스위치를 누름: 내장 LED가 완전히 꺼짐
- 손을 뗌: 내장 LED가 다시 켜짐

이 테스트를 먼저 하는 이유는 다음과 같습니다.

- 납땜 불량 여부를 바로 확인할 수 있음
- 잭 극성이 뒤바뀌지 않았는지 확인할 수 있음
- BLE 문제와 하드웨어 문제를 분리해서 디버깅할 수 있음

### 2. 하드웨어 테스트 절차

1. 보드에 `InfiniteTap_HardwareTest` 업로드
2. USB-C로 전원 공급
3. 외부 적응형 스위치를 3.5mm 잭에 연결
4. LED 상태 확인

정상 동작 기준:

- 스위치를 누르지 않았을 때 LED가 켜져 있어야 함
- 스위치를 누르는 동안 LED가 꺼져 있어야 함
- 손을 떼면 LED가 다시 켜져야 함

정상 동작하지 않는다면 아래를 우선 점검하세요.

- `GND`와 `GPIO 8` 배선이 맞는지
- PJ-320에서 사용하지 않는 중간 핀이 제거되었는지
- 잭 소켓의 `Tip`와 `GND`를 반대로 연결하지 않았는지
- 스위치 자체가 `눌렀을 때 쇼트(close)` 되는 방식인지

## BLE 펌웨어 사용법

하드웨어 테스트가 끝나면 `InfiniteTap_BLE/InfiniteTap_BLE.ino`를 업로드해 BLE 동작을 시험할 수 있습니다.

이 스케치는 다음 방향을 기준으로 작성되어 있습니다.

- BLE HID 키보드처럼 동작
- 스위치 입력을 키 이벤트로 전송
- 내장 RGB LED로 광고/연결/이벤트 상태 표시
- 중앙 버튼 롱프레스로 본딩 정보 초기화

### 현재 BLE 스케치의 의도된 상태 표시

- `파란색 깜빡임`: 광고 중, 페어링 또는 재연결 대기
- `초록색 켜짐`: BLE 연결 완료
- `빨간색 잠깐 점등`: 스위치 이벤트 발생
- `중앙 버튼 3초 이상 누름`: 본딩 삭제 후 재시작

### 입력 사용 개념

BLE 스케치는 단일 스위치 하나만으로도 여러 입력을 구분하려는 방향을 포함합니다.

- 짧은 1회 입력
- 1회 길게 누르기
- 2회 입력

이 구조는 Vision Pro의 스위치 제어에서 서로 다른 동작을 매핑하기 위한 기반입니다.

## Vision Pro에서의 사용 흐름 예시

1. InfiniteTap 전원 연결
2. BLE 광고 시작 확인
3. Vision Pro의 Bluetooth 또는 접근성 관련 설정에서 기기 페어링
4. `Switch Control`에서 해당 입력을 스위치로 등록
5. 원하는 동작에 키 입력을 매핑

실사용 시에는 아래 항목이 중요합니다.

- 연결이 끊겼을 때 빠르게 재광고할 것
- Vision Pro가 깨어났을 때 가능한 빠르게 재연결될 것
- 입력 인식 중 `delay()`에 의존하지 않고 비차단 방식으로 동작할 것

## 추천 검증 순서

1. `InfiniteTap_HardwareTest`로 배선 검증
2. 스위치를 여러 번 눌러 접점 안정성 확인
3. `InfiniteTap_BLE` 업로드
4. BLE 광고 LED 패턴 확인
5. Vision Pro 또는 테스트용 BLE 호스트와 페어링
6. 실제 스위치 제어 입력으로 동작 확인

## 주의사항

- 이 프로젝트는 **배터리 없이 USB-C 상시 전원**을 전제로 합니다.
- GPIO 8 입력은 기본적으로 풀업을 사용하므로, 스위치를 누르면 `GND`로 떨어지는 구조여야 합니다.
- M5Stamp C3U는 작고 밀집된 보드이므로 납땜 브리지와 단락을 특히 주의해야 합니다.
- 중앙 버튼을 본딩 삭제 기능에 사용할 경우, 실수로 초기화되지 않도록 롱프레스 시간을 충분히 두는 것이 좋습니다.

## 트러블슈팅

### LED가 항상 꺼져 있음

- `GPIO 8`이 계속 `LOW`로 끌려가는지 확인하세요.
- 잭 소켓 배선이 반대로 연결되었는지 확인하세요.
- 납땜 브리지로 `G8`과 `GND`가 붙어 있지 않은지 확인하세요.

### 스위치를 눌러도 반응이 없음

- 외부 스위치가 실제로 접점을 닫는 타입인지 확인하세요.
- 3.5mm 잭 규격과 소켓 접점 위치를 다시 확인하세요.
- 테스트 스케치부터 다시 업로드해 하드웨어 문제인지 분리하세요.

### BLE 연결이 잘 안 됨

- 먼저 하드웨어 테스트 스케치가 정상인지 확인하세요.
- 사용 중인 `HijelHID_BLEKeyboard` 라이브러리가 현재 ESP32-C3 환경과 맞는지 확인하세요.
- 페어링 기록이 꼬였을 가능성이 있으면 본딩 초기화 기능을 사용해 다시 시도하세요.
